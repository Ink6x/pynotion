# pynotion 開発計画ドキュメント

時給 5,500〜6,000 円帯、最終的に 8,000 円帯を狙えるポートフォリオへ育てるための
開発計画。計画の全体像と各ドキュメントの役割は以下のとおり。

| ドキュメント | 内容 |
|---|---|
| [01-prd.md](./01-prd.md) | 目的・単価帯別ギャップ分析・成功基準 |
| [02-architecture.md](./02-architecture.md) | 目標アーキテクチャと主要な設計判断 |
| [03-roadmap.md](./03-roadmap.md) | Phase 1〜4 のロードマップ(依存関係・リスク・工数) |
| [04-tasks.md](./04-tasks.md) | チェックボックス式タスクリスト(進捗管理はここ) |

## 進捗ダッシュボード

| Phase | 内容 | 狙い | 状態 |
|---|---|---|---|
| 1 | 認証/RBAC + Docker/CI/本番安全化 | 5,500円帯の最低ラインクリア | 完了 (2026-06-07, PR #2〜#9) |
| 2 | PostgreSQL/全文検索 + OpenAPI + E2E + 計測 | 6,000円帯を確実に | 未着手 |
| 3 | ブロックネスト + リアルタイム同期 + 履歴 | 8,000円帯への跳躍 | 未着手 |
| 4 | CRDT共同編集 + DBビュー + 非同期処理 | 8,000円帯を盤石に | 未着手 |

## 運用ルール

- 進捗は [04-tasks.md](./04-tasks.md) のチェックボックスと本ダッシュボードの「状態」列で管理する
- 状態は `未着手` → `進行中` → `完了` の3値
- 各機能は `feat/<機能名>` ブランチで実装し、PR で main へマージする
- 設計判断を変更した場合は [02-architecture.md](./02-architecture.md) に追記する(決定の上書きではなく履歴として残す)
